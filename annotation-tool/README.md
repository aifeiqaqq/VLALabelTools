## 快速开始

### 1. 启动工具

```bash
cd annotation-tool
npm install
npm run dev
```

打开浏览器访问显示的地址（通常是 `http://localhost:5173`）

### 2. 创建第一个项目

首次打开会看到项目列表页面，点击「新建项目」：

| 字段 | 说明 | 示例 |
|------|------|------|
| **视频文件** | 选择要标注的MP4视频 | `task_drawer_001.mp4` |
| **标注员 ID** | 您的身份标识 | `annotator_01` |
| **场景 ID** | 拍摄场景标识 | `lab_001` |
| **任务类型** | 选择任务类型 | 抽屉任务 / 咖啡机任务 |

点击「开始标注」进入标注界面。

## 📍 核心功能：标注关键帧

### 什么是关键帧？

关键帧是任务执行过程中**状态发生变化的时刻**。例如：
- 机器人手接触到物体
- 抽屉从关闭变为打开
- 物体从桌上被拿起

### 标注流程

#### 1. 找到关键帧
- 播放视频或使用方向键逐帧浏览
- 找到状态变化的准确时刻
- 按 `Space` 暂停

#### 2. 按 `M` 键打标
弹出标注对话框，显示当前帧的缩略图和信息

#### 3. 选择节点模式

**新建节点** vs **复用节点**：

| 模式 | 适用场景 | 示例 |
|------|---------|------|
| 新建节点 | 首次出现的状态 | 第一次看到"抽屉打开" |
| 复用节点 | 之前已标注过的状态 | 再次遇到"抽屉打开" |

💡 **判断标准**：如果Node Meta字段（如drawer_state, gripper_state）完全相同，就复用已有节点。

#### 4. 填写节点信息（新建模式）

**状态描述**（必填）：
- 用一句话描述当前状态
- 例如："抽屉关闭，手空"

**Node Meta**（必填）：
- 根据任务类型填写结构化字段
- 抽屉任务示例：
  - `drawer_state`: closed / half_open / open
  - `gripper_state`: open / closed
  - `object_in_hand`: None / mug / pen

#### 5. 选择父节点（建立连接）

**父节点选项**：
- **无父节点（根节点）**: 这是任务的起始状态
- **选择已有节点**: 从哪个状态转换到当前状态

💡 **支持多分支**：
- 可以有多个起始点（多个根节点）
- 一个状态可以连接到多个后续状态（分支）
- 多个状态可以连接到同一个状态（合并）

#### 6. 填写动作描述（非根节点）

如果选择了父节点，需要描述动作：
- 描述从父节点到当前节点的动作
- 例如："右手握把手向外拉"
- 可以从建议列表选择，也可以输入新动作

#### 7. 确认标注

点击「确认标注」，系统会：
- ✅ 创建或关联节点
- ✅ 创建动作边（如果选择了父节点）
- ✅ 保存关键帧信息
- ✅ 自动保存到本地存储

## 📊 查看和管理

### 右侧面板

#### 统计卡片
显示项目整体统计：
- **节点数**: 创建的状态节点数量
- **边数**: 状态转换（动作）数量
- **帧标记数**: 标注的关键帧总数

#### 节点列表
- 显示所有状态节点
- 🟢 绿色边框 = 最新创建的节点
- 显示Node Meta信息
- 点击节点可编辑

#### 边列表
- 显示所有状态转换
- 格式：`001 → 002 · 动作描述`
- 显示起点、终点和动作

### 已标关键帧列表（左下方）

- 按时间顺序显示所有已标帧
- 包含缩略图、节点ID、帧号、时间戳
- 点击可跳转到该帧
- 🟢 绿色标签 = 最新标注

## 🗂️ 标签页功能

### ◉ Annotate（标注）
主要工作界面，包含：
- 视频播放器
- 播放控制
- 已标帧列表
- 节点和边信息面板

### ⬡ Graph（图结构）
可视化展示整个任务流程：

**节点颜色说明**：
- 🟢 **绿色边框** - 起始节点（根节点）
- 🔵 **青色边框** - 终止节点（无后续状态）
- 🟣 **紫色边框** - 中间节点
- 🟠 **橙色边框** - 当前选中节点
- 🟡 **黄色背景** - 多父节点（合并点）

**特殊标记**：
- "·分支" - 该节点有多个后续状态
- 帧数统计 - 显示每个节点被标注的次数

**导出图结构**：
- 点击顶部「📊 导出 Graph」按钮
- 导出高质量SVG矢量图
- 可用于论文、报告、演示

### ⊞ Library（动作库）
- 查看所有已使用的动作
- 按使用次数排序
- 搜索动作描述

## ✏️ 编辑和删除

### 编辑节点

1. 在「已标帧列表」中点击要修改的帧
2. 或在Graph标签页点击节点
3. 弹出编辑对话框
4. 修改状态描述或Node Meta
5. 点击「确认修改」

⚠️ **注意**：修改节点会影响所有引用该节点的边，系统会显示受影响的边列表。

### 删除标记

目前不支持直接删除，如需删除请：
1. 记录需要删除的帧信息
2. 完成标注后导出JSON
3. 手动编辑JSON文件
4. 重新导入

## 💾 导出和备份

### 导出JSON数据

点击顶部「↓ 导出 JSON」按钮：

**导出内容**：
```json
{
  "version": "1.0",
  "session": { ... },
  "statistics": { ... },
  "nodes": [
    {
      "node_id": "001",
      "task_type": "drawer",
      "timestamp": 0,
      "state_description": "抽屉关闭",
      "parent_node": null,
      "node_meta": { ... }
    }
  ],
  "edges": [
    {
      "from_node_id": "001",
      "to_node_id": "002",
      "process_description": "opening"
    }
  ],
  "process_library": [ ... ]
}
```

**文件命名**：
- 自动命名：`{视频名}_{时间戳}.json`
- 例如：`task_drawer_001_2026-03-09_14-30-00.json`

### 导出Graph图像

点击顶部「📊 导出 Graph」按钮：
- 导出SVG矢量图格式
- 文件名：`graph_{标注员ID}_{场景ID}_{时间戳}.svg`
- 可在Illustrator、浏览器中打开
- 无损缩放，适合插入文档

### 备份建议

⚠️ **强烈建议定期备份**：

1. **每日工作结束**：导出JSON备份
2. **完成重要标注后**：立即导出
3. **更换电脑前**：导出所有项目
4. **清理浏览器前**：务必先导出

## 📝 标注规范

### 节点复用判断

**✅ 应该复用已有节点**：
- Node Meta枚举字段完全相同
- 视觉状态完全一致
- 语义相同（允许描述略有不同）

**❌ 应该新建节点**：
- 任何Node Meta字段不同
- 物体位置或状态发生变化
- 首次出现的状态

### 关键帧选择标准

**推荐标注的时刻**：
- ✅ 任务起始状态
- ✅ 动作开始前的准备状态
- ✅ 动作完成后的结果状态
- ✅ 状态发生明显变化的时刻
- ✅ 分支点（可能有多种执行路径）

**不推荐标注**：
- ❌ 动作执行过程中的中间帧
- ❌ 状态没有变化的帧
- ❌ 过于相似的连续帧

### 状态描述规范

**推荐写法**：
- ✅ "抽屉关闭，手空"
- ✅ "手握把手，抽屉半开"
- ✅ "抽屉完全打开，手放开"

**不推荐**：
- ❌ "1"（过于简略）
- ❌ "机器人正在执行拉抽屉的动作"（过程而非状态）
- ❌ "第15帧"（使用帧号）

### 动作描述规范

**推荐写法**：
- ✅ "右手握把手"
- ✅ "向外拉抽屉"
- ✅ "放开把手"

**不推荐**：
- ❌ "opening"（使用中文）
- ❌ "做了一些操作"（不明确）

## ❓ 常见问题

### 视频相关

**Q: 支持哪些视频格式？**
A: 推荐使用MP4（H.264编码）。大多数浏览器支持的格式都可以，但MP4兼容性最好。

**Q: 视频无法播放？**
A:
1. 确认视频格式是MP4
2. 尝试刷新页面
3. 检查视频文件是否损坏

**Q: 视频很卡怎么办？**
A:
- 大于30分钟的视频可能影响性能
- 建议将长视频分段处理
- 关闭其他浏览器标签页

### 标注相关

**Q: 按M键没反应？**
A:
1. 确认视频已加载完成
2. 确认没有其他对话框打开
3. 确认焦点在页面上（点击一下页面）

**Q: 如何修改已标注的节点？**
A: 点击左下方「已标帧列表」中的帧缩略图，或在Graph标签页点击节点。

**Q: 如何删除错误的标注？**
A: 目前不支持UI删除，需要导出JSON后手动编辑。

**Q: 可以标注多个视频吗？**
A: 可以！一个项目支持多个相关视频，所有视频共享节点定义和动作库。

### 数据相关

**Q: 数据保存在哪里？**
A: 保存在浏览器的IndexedDB（元数据）和OPFS（视频文件）中，不会上传到服务器。

**Q: 清除浏览器缓存会丢失数据吗？**
A: 是的！清除浏览器数据会删除所有项目。请务必先导出备份。

**Q: 可以在不同电脑上使用吗？**
A: 数据存储在本地浏览器中。如需更换电脑：
1. 在旧电脑导出项目
2. 在新电脑导入项目
3. 重新关联视频文件

**Q: 存储空间不足怎么办？**
A:
1. 导出完成的项目
2. 删除已备份的项目
3. 浏览器存储空间通常为磁盘的60%

### 导出相关

**Q: 导出的JSON文件包含什么？**
A: 包含所有标注数据：节点、边、统计信息、动作库。不包含视频文件本身。

**Q: 导出的SVG图像是什么？**
A: 高质量矢量图，展示任务流程图结构，可用于论文、报告、演示。

**Q: 如何备份项目？**
A: 在项目列表页面点击项目卡片的「导出」按钮，保存JSON文件。

## 🔧 故障排除

### 启动失败

```bash
# 确保已安装依赖
npm install

# 清除缓存重试
rm -rf node_modules
npm install
npm run dev
```

### 页面空白

1. 检查浏览器Console是否有错误
2. 尝试使用Chrome/Edge最新版
3. 清除浏览器缓存并刷新

### 自动保存失败

1. 检查浏览器存储空间
2. 查看Console错误信息
3. 手动点击导出备份数据

### 导入项目失败

1. 确认JSON文件格式正确
2. 检查文件是否完整（未截断）
3. 尝试用文本编辑器打开查看
