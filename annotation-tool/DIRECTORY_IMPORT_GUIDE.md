# 目录批量导入功能使用指南

## 功能概述

annotation-tool 现已支持**目录批量导入**功能，允许一次性导入整个文件夹下的所有视频文件。

### 核心特性

✅ **双导入模式** - 支持单个视频和目录批量导入两种方式
✅ **递归扫描** - 自动扫描目录及所有子目录中的视频文件
✅ **统一配置** - 一次设置应用到所有视频（annotatorId、sceneId、taskType）
✅ **智能过滤** - 自动识别和过滤视频文件，跳过无效文件
✅ **实时进度** - 显示当前处理的视频和总体进度
✅ **错误容错** - 部分视频失败不影响其他视频，最后显示汇总
✅ **向后兼容** - 单文件导入功能保持不变

## 使用方法

### 1. 启动应用

```bash
cd annotation-tool
npm run dev
```

访问 http://localhost:5174

### 2. 创建新项目

在项目列表页面点击"新建项目"按钮。

### 3. 选择导入方式

在"视频文件"区域，你会看到两个按钮：

- **📄 选择单个视频** - 传统的单文件导入
- **📁 选择目录** - 新的批量导入功能

### 4. 目录批量导入步骤

#### 步骤1：点击"选择目录"按钮

浏览器会打开文件夹选择对话框（而不是文件选择）。

#### 步骤2：选择包含视频的文件夹

- 支持选择任意深度的文件夹
- 会自动递归扫描所有子文件夹
- 自动识别视频文件（mp4、webm、mov等）

#### 步骤3：预览将要导入的视频

选择目录后，界面会显示：
```
已选择 X 个视频

✓ video1.mp4 (125.3 MB)
✓ video2.mp4 (98.7 MB)
✓ subdir/video3.mp4 (156.2 MB)
...
```

如果有无效文件，会在下方显示警告：
```
以下文件将被跳过：
• large_file.mp4: 文件过大（超过2GB）
• document.pdf: 不是视频文件
```

#### 步骤4：填写项目信息

- **标注员 ID** - 例如：annotator_01
- **场景 ID** - 例如：lab_001
- **任务类型** - 选择：抽屉 / 咖啡机 / 水龙头

这些设置将应用到所有导入的视频。

#### 步骤5：开始导入

点击"开始标注"按钮，系统会：

1. 创建单个项目
2. 顺序处理每个视频文件
3. 显示实时进度：

```
正在处理视频 3 / 10
video3.mp4
[进度条: 67%]
总进度: 23%
```

#### 步骤6：查看导入结果

- 如果所有视频都成功，直接进入标注页面
- 如果部分视频失败，会弹出汇总：
  ```
  成功导入 8 个视频，2 个失败：
  - corrupted.mp4: 提取元数据失败
  - invalid.mp4: 文件损坏
  ```

## 技术说明

### 文件结构

```
annotation-tool/src/
├── utils/
│   └── batchImport.js          # 新增：批量导入工具函数
└── pages/
    └── SetupPage.jsx            # 修改：添加目录导入UI和逻辑
```

### 关键函数

#### batchImport.js

- `validateVideoFile(file)` - 验证单个视频文件
- `filterVideoFiles(fileList)` - 从文件列表中过滤出视频
- `processSingleVideo(file, projectId, videoIndex, onProgress)` - 处理单个视频的核心函数

#### SetupPage.jsx

- `handleDirectoryChange()` - 目录选择处理器
- `handleBatchStart()` - 批量导入主逻辑
- `handleSingleStart()` - 单文件导入（重构后）
- `handleStart()` - 统一入口，根据模式选择处理方式

### 数据流程

```
用户选择目录
    ↓
filterVideoFiles() 过滤视频文件
    ↓
显示预览列表
    ↓
用户点击"开始标注"
    ↓
创建单个项目 (saveProject)
    ↓
顺序处理每个视频:
  - saveVideoFile() → OPFS存储
  - extractVideoMetadata() → 读取时长/分辨率
  - saveVideo() → IndexedDB存储元数据
  - addVideo() → 添加到VideoStore
    ↓
切换到第一个视频
    ↓
进入标注页面
```

## 支持的视频格式

以下格式的视频文件会被自动识别：

- MP4 (.mp4)
- WebM (.webm)
- MOV (.mov)
- AVI (.avi)
- MKV (.mkv)
- 其他浏览器原生支持的视频格式

### 限制条件

- 单个视频文件不超过 2GB
- 视频必须是浏览器可播放的格式
- 建议使用 H.264 编码的 MP4 文件

## 使用场景示例

### 场景1：批量标注实验视频

假设你有以下目录结构：
```
experiment_2024/
├── trial_01.mp4
├── trial_02.mp4
├── trial_03.mp4
└── camera_angles/
    ├── front_view.mp4
    └── side_view.mp4
```

选择 `experiment_2024` 文件夹后：
- 会导入所有 5 个视频文件
- 包括 `camera_angles` 子文件夹中的视频
- 所有视频共享同一个项目和标注配置

### 场景2：多机位视频标注

```
drawer_task/
├── camera1/
│   ├── take1.mp4
│   └── take2.mp4
├── camera2/
│   ├── take1.mp4
│   └── take2.mp4
└── camera3/
    ├── take1.mp4
    └── take2.mp4
```

选择 `drawer_task` 文件夹后：
- 导入全部 6 个视频
- 按文件名排序：camera1/take1.mp4, camera1/take2.mp4, ...
- 可以在标注页面切换不同视频进行标注
- 节点（状态）在所有视频间共享，便于复用

## 常见问题

### Q1: 目录中有非视频文件会怎么样？

**A:** 非视频文件会被自动跳过。如果文件后缀是视频格式但内容无效，会在预览时显示警告，不影响其他视频导入。

### Q2: 如果部分视频导入失败怎么办？

**A:** 系统会继续处理其他视频，最后显示成功和失败的汇总。只要至少有一个视频成功，项目就会创建。

### Q3: 可以导入多少个视频？

**A:** 理论上没有数量限制，但考虑到浏览器性能和存储空间：
- 建议一次导入不超过 50 个视频
- 总大小取决于浏览器可用存储空间（通常是磁盘的60%）

### Q4: 导入过程中可以取消吗？

**A:** 当前版本不支持取消。建议导入前仔细检查预览列表。（未来版本会添加取消功能）

### Q5: 单文件导入还能用吗？

**A:** 完全可以！单文件导入功能保持不变，只是多了一个目录导入选项。

### Q6: Safari浏览器支持吗？

**A:** 目录选择功能在 Safari 中可能受限。建议使用 Chrome、Edge 或 Firefox 浏览器。

## 性能优化

系统已经内置了多项优化：

✅ **顺序处理** - 避免同时处理多个视频导致内存溢出
✅ **分块写入** - 使用 1MB 分块写入 OPFS，提高大文件性能
✅ **进度节流** - 进度回调限制在 100ms 更新一次
✅ **RAF yield** - 定期释放主线程，防止 UI 卡顿
✅ **智能排序** - 按文件名排序保证导入顺序一致
