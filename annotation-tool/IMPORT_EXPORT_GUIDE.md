# 项目导入/导出指南

## 📤 导出项目（备份）

### 方法 1：在项目列表页导出（推荐）

1. 回到项目列表页（点击左上角的「← 返回」）
2. 找到你要备份的项目
3. 点击项目右侧的 **「↓ 导出备份」** 按钮
4. 选择保存位置，下载 `.json` 文件

### 方法 2：在标注页面导出

1. 打开项目进入标注页面
2. 点击顶部工具栏的 **「📦 详细版 JSON」** 按钮
3. 下载包含完整标注数据的 JSON 文件

## 📥 导入项目（恢复）

### 正确的 JSON 格式

导入的 JSON 文件必须是以下格式：

```json
{
  "version": "2.0",
  "exportedAt": "2026-03-12T10:30:00.000Z",
  "project": {
    "id": "proj_1234567890_xxx",
    "annotatorId": "user_01",
    "taskType": "drawer",
    "sceneId": "lab_001",
    "videoName": "video.mp4",
    "createdAt": "2026-03-10T08:00:00.000Z",
    "updatedAt": "2026-03-12T10:00:00.000Z"
  },
  "videos": [
    {
      "id": "v1_abc123",
      "projectId": "proj_1234567890_xxx",
      "name": "video1.mp4",
      "fps": 30,
      "totalFrames": 300,
      "duration": 10.0,
      "width": 1920,
      "height": 1080,
      "createdAt": "2026-03-10T08:00:00.000Z"
    }
  ],
  "annotations": {
    "nodes": {
      "001": {
        "node_id": "001",
        "state_description": "抽屉打开",
        "actions": [...],
        "node_meta": {...},
        "video_segments": {...},
        "task_type": "drawer",
        "annotator_id": "user_01",
        "created_at": "...",
        "updated_at": "..."
      }
    },
    "marks": {...},
    "actionLib": {...}
  }
}
```

### 导入步骤

1. 在项目列表页，点击 **「↓ 导入项目」** 按钮
2. 选择之前导出的 `.json` 文件
3. 导入成功后，新项目会显示为 `原项目名_imported_时间戳`

## ❌ 常见错误

### 错误 1："无效的项目数据"

**原因：**
- 导入的不是标注工具导出的 JSON 文件
- JSON 文件格式不正确
- 文件损坏

**解决：**
- 确保使用的是通过「导出备份」或「详细版 JSON」按钮生成的文件
- 不要手动编辑导出的 JSON 文件

### 错误 2：导入后没有视频

**原因：**
- 导入的项目只包含标注数据，不包含视频文件本身
- 这是正常的安全设计

**解决：**
- 导入后打开项目
- 系统会提示「需要重新选择视频文件」
- 选择对应的视频文件即可

### 错误 3：导入后标注数据丢失

**原因：**
- 视频文件名不匹配
- 视频内容发生了变化（时长、帧率等）

**解决：**
- 确保使用与导出时完全相同的视频文件
- 如果视频内容有变化，标注可能无法对齐

---

## ⚠️ 重要提醒

1. **视频文件不包含在 JSON 中**
   - JSON 只包含标注数据和项目元信息
   - 视频文件需要单独备份
   - 导入时需要重新选择视频文件

2. **跨设备导入**
   - 在新设备上导入项目时，需要重新选择视频文件
   - 标注数据会保留，但需要重新关联视频

3. **格式兼容性**
   - 不同版本的标注工具可能有格式差异
   - 建议使用相同版本的工具进行导入导出

---

## 🔍 如何验证 JSON 文件

如果你不确定 JSON 文件是否正确，可以：

1. 用文本编辑器打开 JSON 文件
2. 检查是否包含以下关键字段：
   - `project.id`
   - `project.annotatorId`
   - `videos` (数组)
   - `annotations.nodes`

3. 如果不包含这些字段，说明不是正确的备份文件
